match 함수를 사용하여 정규표현식으로 주소를 반환해 a 태그에 넣어주고 주소가 아닌 텍스트들은 그냥 리턴하도록 했다.
```TS
    <p>
      {words.map((word) => {
        return word.match(urlRegex) ? (
          <a href={word} target="_blank">
            {word}
          </a>
        ) : (
          word + ' '
        );
      })}
    </p>
```

2️⃣ graphQL 간략히 공부 (공식문서 + 개발 문서 읽으면서 코드 읽음)

GraphQL = 
RestAPI 의  문제점 over-fetching, under-fetching 을 해결했다.

Rest api 와는 다르게 단일한 endpoint를 가지고 있고 필요한 data 의 scheme 를 정의해서 딱 그것만 요청하고 주고받음 
data 의 shape 정의하는 것, arguments 전달 형식은 다음과 같다. 
#### Type 
```JS
const typeDefs = gql`

type Tweet{
    id: ID
    text:String
    author: User
}

type Query {
    allTweets: [Tweet]
    tweet(id:ID) :Tweet
}
`
```

allTweets 처럼 모든 값들은 Tweet의 배열로 모든 트윗 값을 요청 가능하며 
개별 트윗 (tweet) 은 arguments id 를 받아 Tweet 을 반환한다.

#### Mutation Type
